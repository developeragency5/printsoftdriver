Driver Store Cleanup

Cleaning Up the Windows Driver Store Without Breaking Anything

Over years of updates the Windows driver store can grow into many gigabytes of stored versions. A Friendly Walkthrough to trimming it safely.

Home › Driver Concepts › Driver Store Cleanup

Why It Grows

Every driver update Windows applies leaves the previous version in the driver store, ready in case rollback is needed. After a few years on the same install this can comfortably reach 5–10GB on a typical PC, and quietly more on workstations with many devices.

On large drives this is not a problem; on small SSDs it can quickly become annoying. Windows itself does not aggressively clean these older copies, leaving the choice to the user.

The Built-In Tools

Disk Cleanup includes a 'Device driver packages' option that removes superseded drivers safely. Run it as Administrator (the option only appears in admin mode) and tick that box to claw back disk space.

From the command line, 'pnputil /enum-drivers' lists every third-party driver, and 'pnputil /delete-driver oemNN.inf /uninstall' removes a specific package. Combined with a script, this lets advanced users keep only the latest versions of each driver.
