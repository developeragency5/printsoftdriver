DRIVER STORE EXPLAINED

The Windows Driver Store, Where All Your Drivers Quietly Live

A short tour of the protected folder that Windows uses to keep every driver it has ever installed on your machine.

Home › Driver Concepts › Driver Store Explained

What the Driver Store Is

Windows keeps every driver it installs inside C:\Windows\System32\DriverStore\FileRepository. Each driver lives in its own subfolder, named after the INF file plus a hash, so versions and updates do not overwrite each other.

The store is protected by Windows itself — only the operating system can write to it. This guarantees that every installed driver is intact and its signature is preserved exactly as it was at install time.

Why It Grows Over Time

Every driver update Windows applies leaves the previous version behind in the driver store, ready in case you need to roll back. Over a few years this can quietly accumulate to several gigabytes — usually fine, occasionally inconvenient on small SSDs.

the OS vendor provides PnPUtil and the Disk Cleanup tool's 'Device driver packages' option to safely remove obsolete versions. Manual deletion of files in the driver store is a bad idea — Windows tracks them by hash and a corrupt store can prevent driver installs.

Inspecting What You Have Installed

From an elevated command prompt, 'pnputil /enum-drivers' lists every third-party driver currently in the driver store. The output shows publisher, install date, and signature status — useful when chasing down a misbehaving driver.

Built-in the OS vendor drivers do not show up in this list, because they are managed as part of Windows itself rather than as separate packages. The driver store is only for the third-party additions.
