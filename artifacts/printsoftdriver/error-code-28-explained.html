ERROR CODE 28

Device Manager Error Code 28: When the Driver Is Missing

What Code 28 actually means in Windows Device Manager, and the calm steps that almost always resolve it.

Home › Driver Concepts › Error Code 28

What Code 28 Means

Code 28 means Windows recognised your device but has no driver for it. The device is sitting in Device Manager with a yellow triangle, ready to work, just waiting for the right software bridge between it and the operating system.

This is the easiest Device Manager code to fix in most cases, because the cause is straightforward: install the missing driver.

Where to Find the Right Driver

Right-click the device in Device Manager, choose Properties, and check the Details tab → Hardware Ids. The first ID is your most reliable search term — paste it into a search engine alongside 'driver' to find legitimate vendor pages.

Avoid generic 'driver download' sites. Stick to the device manufacturer's own support page, your laptop maker's page, or Microsoft's Update Catalog at catalog.update.microsoft.com.

  • Find the device's Hardware ID in Device Manager
  • Search for that exact ID on the vendor's site
  • Microsoft Update Catalog is a safe fallback

After the Install

After installing the driver, refresh Device Manager. The yellow triangle should disappear and the device should move to its proper category in the tree. If it does not, reboot once and check again — some drivers complete their setup at boot.

If Code 28 persists, double-check that the driver matches your Windows version (32-bit vs 64-bit, Windows 10 vs 11, x64 vs Arm). Mismatched drivers will install but not actually load.
