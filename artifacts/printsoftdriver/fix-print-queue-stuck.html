Fix: Print Queue Stuck

Fix: Print Queue Stuck and Will Not Clear

When jobs pile up in the print queue and refuse to disappear, this calm walkthrough resets things without restarting Windows.

Home › Troubleshooting › Fix: Print Queue Stuck

The Print Spooler Restart

The print queue lives inside the Windows Print Spooler service. When it freezes, no jobs go in or out. Open Services (services.msc), find Print Spooler, right-click and Restart. The queue usually clears within a few seconds.

If Restart hangs, Stop the service first, manually delete every file inside C:\Windows\System32\spool\DEVICES, then Start the service again. This is the nuclear option but it is harmless — only the pending print jobs are removed.

  • Open services.msc
  • Restart the Print Spooler service
  • Manually delete the queue folder if Restart hangs

Why Queues Get Stuck

Print jobs hang when the device reports a transient error (paper out, ink low, network blip) and Windows fails to recover. Each hung job blocks every following job. Clearing one usually clears the cascade.

Some device drivers also struggle with very large jobs. Splitting a 200-page document into smaller batches can prevent recurring hangs on older drivers.

Long-Term Prevention

If your queue gets stuck regularly, update the device driver. Older drivers — especially those for inkjets sold a decade ago — handle modern Windows updates poorly and benefit hugely from refreshed packages.

Also check whether your device has any built-in firmware updates available. A firmware update on the device itself can resolve the underlying communication issues that cause queues to hang.
